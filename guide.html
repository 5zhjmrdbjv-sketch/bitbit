BTC/JPY ダッシュボードの見方

このページは、エントリー判定・根拠・損切り・利確・推奨数量を冷静に読むためのガイドです。

← ダッシュボードへ戻る 実戦チェックへ
注意：このダッシュボードは投資判断を保証するものではありません。必ず損切り・数量・データ状態を確認し、無理なエントリーは避けてください。

まず見る順番

  • ① データ状態 → ② 現在価格 → ③ エントリー判定 → ④ エントリー根拠 → ⑤ スコア内訳 → ⑥ 損切り・利確・推奨数量 → ⑦ チャート・指標 → ⑧ 総括コメント の順で確認します。
  • 慣れるまでは『判定 → 根拠 → 損切り』の3つを中心に見ればOKです。

データ状態の見方

  • CoinGecko / Coincheck / bitFlyer の価格データなら、基本的には現在価格を使えている状態です。
  • 『直近成功データ』『参考値』が出ているときは最新価格ではない可能性があるため、新規エントリー判断は慎重にしてください。

エントリー判定

  • 🟢 強気：条件が比較的そろっている状態。ただし『絶対に買い』ではなく、入る候補として見る段階です。
  • 🟡 検討：悪くはないが決定打が不足。押し目、出来高増加、MA維持、ブレイク定着を待つ場面です。
  • 🔴 見送り：優位性が低い状態。入らない判断ができた時点で十分価値があります。

エントリー根拠の読み方

  • 文章は『今どういう場面か』『好材料』『注意点』の3つに分けて読みます。
  • 好材料が多くても、注意点に『レジスタンスが近い』『RRが悪い』『ボラが高い』がある場合は数量を落とすか見送りを検討します。

スコア内訳の見方

  • MAがプラスなら短期トレンドが上向き、MACDがプラスなら勢いが買い方向です。
  • RSI、出来高、サポレジ、ボラ、RRがマイナスなら、その項目がエントリーの弱点です。特にRRが悪い場合は無理に入らない判断が大切です。

どこで買い、どこで売るか

  • 買う候補は『エントリー』表示、またはサポート/MA付近です。
  • 売る候補は『利確』表示、またはレジスタンス付近です。
  • 逃げる場所は『損切り』表示です。買ってから考えるのではなく、買う前に必ず決めます。

損切り・利確・推奨数量

  • 想定資金と許容損失から、1回あたりの損失上限を決めます。例：100万円の1%なら1万円まで。
  • 推奨数量は『最大でもこのくらいまで』という上限です。最初は表示数量の1/2〜1/3程度から考えると安全です。

買わない条件

  • 🔴 見送り、参考値表示、直近成功データ表示、RRが悪い、RSI過熱、レジスタンス直前、ボラが高すぎる、出来高が弱い場合は基本的に見送り優先です。
  • 迷ったときは入らない。チャンスは何度でも来ますが、資金は守る必要があります。

実戦チェックリスト

  • 1. データがリアルタイムか確認
  • 2. 判定が🟢または🟡か確認
  • 3. エントリー根拠を読む
  • 4. 注意点が致命的でないか確認
  • 5. RRが悪すぎないか確認
  • 6. 損切り位置を確認
  • 7. 推奨数量以下、できれば半分以下で検討
